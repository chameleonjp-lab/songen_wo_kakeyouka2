# 尊厳を賭けようか2 仕様書

更新日: 2026-06-24
対象ファイル: `index.html`  
公開想定URL: `https://chameleonjp.codeberg.page/songen_wo_kakeyouka2/`

この文書は、ブラウザゲーム「尊厳を賭けようか2」の正式な仕様書です。実装を変更する場合は、この仕様と矛盾しないようにします。

---

## 1. ゲーム概要

「尊厳を賭けようか2」は、前作の1戦型バトルを土台にした、最大5戦の連戦ゲームです。

プレイヤーはCPUを順番に倒していきます。第1戦から第4戦までは、勝利後に報酬を選びます。第5戦が終わると、勝敗にかかわらず総合結果へ進みます。

このゲームの中心は、壊れた体を次の戦闘へ持ち越すことです。腕、足、頭が壊れても完全には詰まないように、後ろ腕の前出し、足破壊時の減速移動、両腕なし頭攻撃を許可します。

## 4. 連戦進行仕様

### 4.1 基本進行

最大5戦です。勝利した時だけ次の戦闘へ進めます。

```text
第1戦 勝利 → 報酬選択 → 第2戦
第2戦 勝利 → 報酬選択 → 第3戦
第3戦 勝利 → 報酬選択 → 第4戦
第4戦 勝利 → 報酬選択 → 第5戦
第5戦 終了 → 総合結果
```

### 4.2 結果ごとの扱い

| 1戦の結果 | 連戦上の扱い |
|---|---|
| 勝利 | 第1〜第4戦なら報酬選択へ進む。第5戦なら総合結果へ進む |
| 敗北 | その時点で総合結果へ進む |
| 引き分け | その時点で総合結果へ進む |
| リタイア | その時点で総合結果へ進む |

引き分けは続行不可です。

## 7. 部位仕様

### 7.1 部位一覧

ゲーム上の部位は次です。

```text

leftArm
rightArm
leftFoot
rightFoot
heart
```

心臓は急所です。心臓に攻撃が通ると、その戦闘は決着します。

### 7.2 耐久値

```js
const PART_MAX = Object.freeze({
  leftArm: 2,

```

腕と足は2回で壊れます。頭は3回で壊れます。頭が壊れた場合は、表示を `💩` に変えます。

## 8. 部位持ち越し仕様

### 8.1 持ち越すもの

プレイヤーだけ、次の部位状態を次戦へ持ち越します。

```text
head.damage / head.broken
leftArm.damage / leftArm.broken
rightArm.damage / rightArm.broken
leftFoot.damage / leftFoot.broken
rightFoot.damage / rightFoot.broken
```

### 8.2 持ち越さないもの

次の状態は持ち越しません。

```text
現在位置
向き
ジャンプ状態
しゃがみ状態
ガード状態
スタミナ残量
攻撃待ち時間
ヒット硬直
画面上の攻撃
画面上のエフェクト
CPUの部位状態
CPUの思考状態
```

第2戦以降は、通常の戦闘初期化を行った後、プレイヤーの部位状態だけを `runState.playerCarryParts` で上書きします。

## 9. 報酬仕様

### 9.1 報酬が出る条件

第1戦から第4戦で勝利した場合だけ、報酬選択画面を出します。

第5戦では勝利しても報酬選択を出しません。すぐ総合結果へ進みます。

### 9.2 報酬の種類

報酬は次のどちらかです。

```text
壊れている部位を応急処置（耐久1で復帰）
スコア +1000
```

### 9.3 回復対象

回復できるのは、次のうち `broken === true` の部位だけです。

```text
head
leftArm
rightArm
leftFoot
rightFoot
```

`damage > 0` でも、`broken === false` の部位は回復候補に出しません。

### 9.4 回復処理

応急処置は完全回復ではありません。ダメージを1だけ戻し、耐久1の状態で復帰させます。

```text
頭：damage 3 → 2、broken true → false
腕：damage 2 → 1、broken true → false
足：damage 2 → 1、broken true → false
```

壊れている部位がない場合は、スコア+1000だけ選べます。

## 12. 腕仕様

### 12.1 有効前面腕

相手に近い側の腕を前面腕とします。

前面腕が壊れていて、後ろ腕が残っている場合は、後ろ腕を前へ出します。この腕を有効前面腕とします。

```text
前面腕が生きている → 前面腕が有効前面腕
前面腕が壊れているが後ろ腕が生きている → 後ろ腕が有効前面腕
両腕が壊れている → 有効前面腕なし
```

有効前面腕は、表示、ガード、通常攻撃、被弾判定で共通して使います。

### 12.2 中段攻撃を受ける時

立ち状態で中段攻撃を受ける場合、命中先は次です。

```text
有効前面腕 → なければ心臓
```

両腕が壊れている場合、中段攻撃は心臓へ通ります。

---

## 13. 攻撃仕様

### 13.1 攻撃種別

攻撃は2種類です。

```text
fist: 腕が1本以上ある時の通常攻撃
head: 両腕が壊れている時の頭・💩攻撃
```

### 13.2 通常攻撃

腕が1本以上ある場合、攻撃ボタンで通常攻撃を出します。

```text
立ち: 中段
しゃがみ: 下段
ジャンプ: 上段
```

### 13.3 頭・💩攻撃

両腕が壊れている場合、攻撃ボタンで頭攻撃を出します。

```text
頭が壊れていない: 選択中の頭で攻撃
頭が壊れている: 💩で攻撃
```

頭・💩攻撃は地上専用です。ジャンプ中は出せません。しゃがみ中でも出せますが、判定は常に中段です。

両腕なしCPUは頭・💩攻撃しかできないため、攻撃直前ジャンプをしません。CPUのランダム思考内ジャンプも、両腕なしでは控えます。

```text
高さ: mid
スタミナ消費: 1.5
射程: LIMITS.fistRange - LIMITS.hitWidth
```

初期値では、通常攻撃射程が168、頭攻撃射程が122です。

頭・💩攻撃も攻撃回数に含めます。

## 14. 命中仕様

### 14.1 相手が立ち状態

```text
上段 → 頭
中段 → 有効前面腕。なければ心臓
下段 → 近い足。足がなければ空振り
```

### 14.2 相手がしゃがみ状態

```text
上段 → 空振り
中段 → 頭
下段 → 近い腕または足。なければ心臓
```

### 14.3 相手がジャンプ状態

```text
足があれば足
足がなければ空振り
```

## 15. ガード仕様

ガードには有効前面腕が必要です。

```text
中段ガード → 中段攻撃を防ぐ
下段ガード → 下段攻撃を防ぐ
上段攻撃 → ガードではなく、しゃがみ回避の対象
```

頭・💩攻撃は中段攻撃なので、中段ガードで防げます。

ガードで頭・💩攻撃を防いだ場合、攻撃側の頭にはダメージを入れません。頭にダメージが入るのは、拳攻撃または頭攻撃と相殺した場合だけです。

ガード中は、プレイヤーもCPUも攻撃・ジャンプ・移動ができません。しゃがみ姿勢を保つことはできます。

CPUが下段攻撃をガードする場合、`crouchUntil` を `guardMs` 分延長し、ガード時間中にしゃがみがすぐ解除されないようにします。

## 16. 攻撃相殺仕様

### 16.1 相殺対象

```text
fist vs fist: 相殺する
head vs fist: 相殺する
head vs head: 相殺する
```

### 16.2 相殺条件

次の条件を満たす場合、相殺します。

```text
攻撃者が違う
両方とも removed / resolved ではない
X座標差が28px以内
fist vs fist は height が同じ
head vs fist は head が mid、fist も mid
head vs head は両方とも mid
```

相殺後は両方の攻撃を消します。同じフレームで同じ攻撃が複数回相殺されないよう、相殺後はその組の処理を抜けます。

### 16.3 head vs fist の追加処理

頭・💩攻撃が拳攻撃と相殺した場合、頭攻撃を出した側に追加処理を行います。head vs head の場合は、双方の頭攻撃に同じ追加処理を行います。

#### 頭が壊れていない場合

頭に1ダメージを入れます。これで頭が壊れた場合、通常の頭破壊として集計します。

#### 頭が壊れている場合

💩攻撃が相殺された扱いです。

```text
プレイヤーの💩攻撃が相殺 → playerBrokenPartHits +1、playerPoopAttackClashes +1
CPUの💩攻撃が相殺 → enemyBrokenPartHits +1、cpuPoopAttackClashes +1
```

プレイヤーの💩攻撃が叩かれた場合、スコア内訳では「壊れた自分頭への追加被弾」として -300 相当になります。

CPUの💩攻撃を叩いた場合、スコア内訳では「壊れた敵頭への追加攻撃」として +300 相当になります。

💩攻撃が相殺された場合は、頭攻撃を出した側に短い無敵時間を付けます。すでに無敵時間中の場合、相殺した攻撃は消えますが、追加被弾回数と -300 / +300 相当のスコア用統計は重ねて加算しません。

## 17. 勝敗判定

心臓に攻撃が通った時点で、その戦闘は決着します。

```text
プレイヤーがCPUの心臓へ攻撃を通す → 勝利
CPUがプレイヤーの心臓へ攻撃を通す → 敗北
同一フレームで両方の心臓に攻撃が通る → 引き分け
リタイアボタンを押す → リタイア
制限時間が切れる → 引き分け
```

## 18. スコア仕様

### 18.1 1戦ごとのスコア

1戦ごとのスコア計算式は次です。

```text
勝利ボーナス: +5000
攻撃回数: -30 × attacks
ガード成功: +120 × guardSuccess
敵通常部位破壊: +300 × (enemyPartsBroken - enemyHeadBroken)
敵頭破壊: +1000 × enemyHeadBroken
壊れた敵頭への追加攻撃: +300 × enemyBrokenPartHits
自分部位破壊: -500 × playerPartsBroken
壊れた自分頭への追加被弾: -300 × playerBrokenPartHits
経過秒数: -10 × elapsedSec
```

この式は `calcScore()` と一致させます。

### 18.2 頭攻撃とスコア

頭・💩攻撃も `attacks` に含めます。したがって、1回ごとに -30 の対象です。

### 18.3 総合スコア

総合スコアは次です。

```text
総合スコア = 各戦闘スコア合計 + 報酬スコア加算合計
```

## 23. 固定解釈

この仕様では、次の解釈を固定します。

```text
回復できるのは broken === true の部位だけ
負傷中だが壊れていない部位は回復できない
第5戦後は報酬選択を出さない
引き分けは連戦終了
ランキング送信は総合スコアのみ
頭・💩攻撃は地上専用
頭・💩攻撃は常に中段
head vs head は相殺する
💩攻撃、頭攻撃、拳攻撃の相殺処理は、例外が発生してもゲームループを停止させない
相殺処理で例外が発生した場合、該当projectileを削除して戦闘を継続する
無敵時間中の頭・💩相殺では、追加ダメージや追加ペナルティを重ねない
head vs fist / head vs head の相殺処理は防御的に実装する
ガードで頭・💩攻撃を止めても、攻撃側の頭にダメージは入らない
💩攻撃を叩かれた時は、壊れた頭への追加被弾として扱う
CPUにも欠損時の継戦ロジックを適用する
```

## 24. 受け入れ条件

次を満たせば、この仕様の実装として成立します。

```text
ゲーム名、slug、URLが2用になっている
最大5戦の連戦として遊べる
勝利時のみ次戦へ進める
第1〜第4戦の勝利後に報酬選択が出る
第5戦後は報酬選択なしで総合結果へ進む
CPU名入力欄が存在しない
敵名が固定順で出る
プレイヤーの壊れた部位だけが次戦へ持ち越される
報酬で壊れた部位を応急処置（耐久1で復帰）できる
報酬でスコア+1000を選べる
```

## 追補: 2026-06-24 CPU難易度・報酬・結果画面アップデート

### CPU戦闘パターン

CPUは各戦闘開始時に、次の5種類の戦闘パターンからランダムに1つ設定します。第1戦は強すぎる偏りを避けるため、ガード反撃型・足狙い型・ラッシュ型から選びます。第5戦はガード反撃型をベースにした強めの最終戦設定を使います。

| パターン | 主な特徴 |
|---|---|
| ガード反撃型 | 高い防御確率とガード後反撃を重視 |
| 足狙い型 | 下段攻撃の比率が高く、足破壊を狙う |
| ラッシュ型 | 攻撃頻度が高く、守りは薄め |
| 頭狙い型 | 上段攻撃の比率が高く、頭破壊を狙う |
| 待ち・差し返し型 | 距離を取りながら防御と差し返しを狙う |

各戦闘レポートには `cpuProfileLabel` を保存し、詳細レポートにもCPU戦闘パターン名を表示します。検証用にCPU攻撃回数、CPU頭攻撃回数、CPUガード成功数、CPUが出した下段・中段・上段回数も記録します。

### 戦闘番号によるCPU強化補正

戦闘番号ごとにCPUの攻撃・防御系パラメータを補正します。補正対象は `attackChance`、`partDefenseChance`、`heartDefenseChance`、`counterAfterGuardChance` です。移動速度、部位耐久、スコア計算式は補正しません。

| 戦闘番号 | 補正倍率 |
|---:|---:|
| 第1戦 | 0.85 |
| 第2戦 | 1.00 |
| 第3戦 | 1.15 |
| 第4戦 | 1.30 |
| 第5戦 | 1.50 |

上限値は `attackChance <= 0.72`、`partDefenseChance <= 0.78`、`heartDefenseChance <= 0.95`、`counterAfterGuardChance <= 0.85` です。CPUはプロファイルの `lowBias` / `highBias` とプレイヤーの負傷状態を見て、攻撃直前に下段・中段・上段の姿勢を選びます。プレイヤーの足・頭・腕の負傷やスタミナ不足は、CPUの狙いに小さな補正だけを与えます。

### 報酬仕様

第1戦〜第4戦の勝利後報酬は、次の3種類です。

| 報酬 | 効果 |
|---|---|
| 負傷部位+1回復 | `damage > 0` かつ `broken === false` の部位の `damage` を1減らす |
| 壊れた部位の復活 | `broken === true` の部位を `broken = false` にし、`damage = PART_MAX[part] - 1` にする |
| スコア+1000 | 総合スコアへ `REWARD_SCORE` を加算する |

復活は完全回復ではありません。復活した部位は耐久1で使える状態に戻ります。腕・足は `damage = 1, broken = false`、頭は `damage = 2, broken = false` になります。

### 結果画面とランキング

結果画面には、従来の「総合結果をシェア」「詳細レポートをコピー」「もう一度遊ぶ」「他のゲームで遊ぶ」に加えて、「ホームへ戻る」ボタンを表示します。ホームへ戻る時は戦闘状態を止め、`projectile` と `float-fx` を削除し、`playing-lock` を解除してホーム画面を表示します。

結果画面下部には「スコアランキング TOP10」を表示します。総合スコア送信後、既存のSupabase URLとPublishable keyを使い、`/rest/v1/rpc/get_best_score_ranking` へ `p_game_slug` と `p_limit: 10` を送って取得します。`p_limit` 付き取得に失敗した場合は `p_game_slug` のみで再試行し、フロント側で上位10件に切り詰めます。送信に失敗してもランキング取得は試み、ランキング取得に失敗した場合は「ランキングを読み込めませんでした」と表示してゲーム全体は止めません。

総合スコア表示は「総合スコア」という語が途中で改行されないよう、ラベルと点数を別要素に分け、`word-break: keep-all` と `white-space: nowrap` を適用します。

### ルール説明

ルール説明は「勝利条件」「操作」「スタミナ」「部位」「報酬」にグループ分けします。スタミナ消費は次の通り明記します。

| 行動 | スタミナ消費 |
|---|---:|
| 🤜攻撃 | 1 |
| 頭・💩攻撃 | 1.5 |
| ガード | 発動時に0.5 |

ガード成功時の追加スタミナ消費はありません。スタミナは通常、一定時間ごとに回復します。💩だけで戦う状態では、スタミナ回復速度が1.5倍になります。報酬説明では、負傷部位の+1回復、壊れた部位の復活、スコア+1000の違いと、復活が完全回復ではないことを明記します。

### ジャンプ・スタミナ・表示位置

ジャンプ滞空時間 `LIMITS.jumpMs` は従来値800msの0.7倍である560msにします。戦闘画面のスタミナ表示はHUDのTIME / SCOREと重ならないよう、上端から約150px下げた位置に表示します。

💩だけ状態は「両腕が壊れている、かつ頭が壊れている」状態です。この状態では攻撃手段が💩攻撃だけになるため、プレイヤーとCPUの両方でスタミナ回復速度を1.5倍にします。
